# Fulfillment Model

Date: 2026-05-25

Purpose: answer the operational question: if a real business offer arrives, who does the work?

## Core answer

The system should only sell work that can be delivered mostly by the AI/operator system with owner approval, or work that can be safely scoped into a diagnostic deliverable.

The owner should not be forced to understand or manually execute technical work.

## Delivery roles

### AI/System does
- Read prospect context
- Classify the opportunity
- Decide whether the request is sellable
- Draft replies
- Build checklists
- Create diagnostic reports
- Prepare workflow maps
- Prepare product listings and affiliate content
- Prepare client-facing deliverables
- Prepare implementation briefs if outside help is needed

### Owner does
- Approves outgoing emails
- Approves paid offers
- Approves submitted applications
- Connects accounts or provides unavoidable access
- Decides whether to accept larger commitments

### External human specialist does, only when approved
- Deep custom coding
- Complex automation implementation
- Client-specific system setup requiring credentials
- Legal, tax, accounting, regulated or high-liability work
- Large design/build work beyond diagnostic scope

## What we can safely sell first

### 1. Affiliate route
No client service delivery required.

Work:
- Apply to affiliate program
- Add product page/listing
- Create content route
- Track link and clicks

Primary worker: AI/System, with owner approval.

### 2. 19 EUR First Signal Check
Small diagnostic only.

Promise:
- one signal path
- one leak note
- one first recovery action
- short next-step suggestion

Primary worker: AI/System.
Owner only approves final delivery.

Do not promise implementation.

### 3. 49 EUR Signal Audit
Wider diagnostic only.

Promise:
- map of inquiry, booking, demo, quote, partner, or affiliate route
- missed follow-up points
- first recovery actions

Primary worker: AI/System.
Owner only approves final delivery.

Do not promise implementation.

### 4. 490 EUR Cloud Command Center Diagnostic
Serious diagnostic package.

Promise:
- workflow map
- revenue/signal gaps
- approval gaps
- safety notes
- seven-day action plan

Primary worker: AI/System.
Owner approval required before accepting.

Implementation is not included unless separately scoped.

## What not to sell yet

Do not sell:
- custom software builds
- guaranteed revenue
- full agency execution
- legal/financial/medical advice
- anything requiring owner to manually learn complex systems
- anything that requires credentials before payment and clear scope

## Offer acceptance rule

If a business says yes, the system must reply with a scoped offer, not vague promises.

Template logic:

1. If they want a small check -> sell First Signal Check.
2. If they want a broader review -> sell Signal Audit.
3. If they want a complete system diagnosis -> sell Cloud Command Center Diagnostic.
4. If they want implementation -> say implementation can be scoped after diagnostic, not before.

## Emergency clarity line

If a real offer arrives, the answer is:

The AI/System prepares and delivers the diagnostic work. The owner approves. Bigger implementation is either not sold yet or outsourced only after approval.

## Operating principle

Do not create a business where the owner becomes the unpaid worker.
Create a business where the system converts signals into narrow, deliverable products.
