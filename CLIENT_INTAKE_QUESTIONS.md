# Client Intake Questions

## Purpose

Collect enough information to deliver the 48-hour diagnostic.

## Questions

1. What tools are currently part of the system?
2. Which workflows matter most?
3. What breaks most often?
4. Where are API keys and ownership tracked?
5. What still depends on one person remembering details?
6. What should be automated but is risky today?
7. What is the main business goal for the next 30 days?
8. What would make this diagnostic successful?

## Required assets

- website or project summary
- list of tools
- workflow notes
- repo links if relevant
- existing docs if relevant

## Rule

Do not request secrets. Ask for structure, not private credentials.
