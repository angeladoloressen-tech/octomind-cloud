# Operator Batch Mode

## User preference

Do not report after every small action.

The operator should execute in batches and report only after a meaningful block of work is complete.

## Default batch size

20 operator steps.

## Reporting rule

After each batch, report briefly:

- what changed
- what was verified
- what is blocked
- what the next batch should do

## No-noise rule

Avoid long status commentary during execution.

Do not ask the human to perform routine work.

Only interrupt early when:

- financial approval is required
- account login or permission is required
- a connector blocks the action
- a safety or legal boundary appears
- the system lacks required access

## Execution style

For revenue work, continue in this priority order:

1. payment product mapping
2. public gate wiring
3. domain and landing structure
4. SignalAuditRequest capture
5. delivery template
6. target vertical pages
7. proof examples
8. compliance-safe inquiry path
9. conversion tracking
10. next paid offer

## Final rule

Less theatre, more completed blocks.
