## A to Z execution ownership

### Strategy

Owner: OctoMind operator

Tasks:

- choose active offer
- define pricing
- choose target verticals
- freeze non-revenue branches

### Public gate

Owner: Base44 operator

Tasks:

- build public pages
- create industry gates
- collect public-safe requests
- avoid cart and checkout unless payment system is intentionally attached

### Memory and governance

Owner: GitHub operator

Tasks:

- store operating documents
- store revenue rules
- store target account strategy
- store compliance notes

### Request data

Owner: Base44 data layer first, Supabase later

Tasks:

- collect SignalAuditRequest records
- keep intake public-safe
- avoid sensitive customer data in public forms

### Payment

Owner: human approval plus payment provider

Tasks:

- create payment link or invoice
- connect bank and tax details
- approve commercial terms

### Delivery

Owner: OctoMind operator with human approval for private data

Tasks:

- review safe intake
- prepare ranked opportunity note
- prepare next-action sequence
- produce first audit result

### Traffic

Owner: inbound engine

Tasks:

- publish search-intent gates
- target automotive, real estate, energy, and agency verticals
- use public proof pages
- avoid bulk unsolicited email

## Execution rule

The human should only approve unavoidable financial or account-access steps.

Everything else should be executed by the system and reported as outcome, blocker, or next autonomous action.

## Current blocker

A domain can be recommended and checked, but registration and payment require account approval.

Payment collection requires a payment provider or invoice workflow.
